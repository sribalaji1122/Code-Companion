========================================
CRAWL / WALK / RUN LOGIC
========================================

----------------------------------------
CURRENT (Year 0 – Baseline)
----------------------------------------

This represents the exact current ecosystem.

Rules:
- Use only selected tools
- Minimal integrations
- CRM-centric execution
- Manual or reactive campaign setup
- No orchestration layer
- No intelligence layer
- No advanced segmentation
- Batch-based flows only

This stage must still follow full 4-lane + bottom structure.

----------------------------------------
CRAWL (Year 1 – Foundation)
----------------------------------------

Goal: Stabilize & connect.

- Fix siloed integrations
- Introduce CDP if missing
- Centralize customer profiles
- Basic segmentation
- Connect data sources to CDP/CRM
- Limited automation
- Mostly batch flows

No orchestration layer yet.
No decisioning engine.

----------------------------------------
WALK (Year 2 – Optimization)
----------------------------------------

Goal: Automate & align.

- Add Marketing Automation
- Add Journey Orchestration
- Introduce Personalization tool
- Behavioral segmentation
- Better integration patterns
- Campaign performance feedback loop

Some automation.
Still limited AI.

----------------------------------------
RUN (Year 3 – Transformation)
----------------------------------------

Goal: Fully optimized ecosystem.

- Real-time event streaming
- Decisioning engine
- AI-driven personalization
- Closed-loop analytics
- Automated suppression & audience sync
- Intelligence layer
- Event-based orchestration

Must visibly look more advanced than Walk.
